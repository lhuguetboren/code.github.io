Practica GETPOST

  1. GET Request (localhost/api/Unityget)

    • Fetches data from the API and displays it in the UI.

  2. POST Request (localhost/api/Unitypost)

    • Sends user input to the API.

  3. Basic UI Elements

    • Input Field (for entering data to send via POST).
    • Buttons (for sending GET and POST requests).
    • Text Area (for displaying received data).

Scene Setup in Unity

  1. Create a new Unity UI Canvas:

    • Right-click in the Hierarchy → UI → Canvas.
    • Set Canvas Scaler to Scale With Screen Size.

  2. Add UI Elements:

    • Text (responseText):
      • Right-click Canvas → UI → Text.
      • Change its name to ResponseText.
      • Set Text to "Response: " (empty by default).
      • Assign it in the APIManager script.
    • InputField (inputField for POST data):
      • Right-click Canvas → UI → Input Field.
      • Rename it to InputField.
      • Assign it in the script.
    • Buttons:
      • GET Button:
        • Right-click Canvas → UI → Button.
        • Rename to GetButton, set text to "GET Data".
        • Add OnClick Event, assign APIManager.GetRequest.
      • POST Button:
        • Right-click Canvas → UI → Button.
        • Rename to PostButton, set text to "POST Data".
        • Add OnClick Event, assign APIManager.PostRequest.

  3. Attach the Script:

    • Create an empty GameObject in the scene and rename it to APIManager.
    • Attach the APIManager script.
    • Drag the InputField and ResponseText into the script fields in the Inspector.

  4. Run & Test:

    • Play the scene and use the UI buttons to send API requests.

Explicación del Diagrama

  1. GET Request:

    • El usuario presiona el botón GET.
    • Se ejecuta GetRequest(), que inicia StartCoroutine(GetData()).
    • UnityWebRequest envía una solicitud GET a localhost/api/Unityget.
    • La API devuelve una respuesta que se muestra en la UI.

  2. POST Request:

    • El usuario ingresa datos y presiona el botón POST.
    • Se ejecuta PostRequest(), que inicia StartCoroutine(PostData()).
    • Se crea un formulario WWWForm con los datos del usuario.
    • UnityWebRequest envía una solicitud POST a localhost/api/Unitypost.
    • La API devuelve una respuesta que se muestra en la UI.

CÓDIGO CSHARP 

using System.Collections;
using UnityEngine;
using UnityEngine.Networking;
using UnityEngine.UI;

public class APIManager : MonoBehaviour
{
    // URLs for GET and POST requests
    public string getUrl = "http://localhost/api/Unityget";
    public string postUrl = "http://localhost/api/Unitypost";
    
    // UI Elements: Input field for POST request and a text field to display responses
    public InputField inputField;
    public Text responseText;
    
    // Method triggered when GET button is clicked
    public void GetRequest()
    {
        Debug.Log("Starting GET request to: " + getUrl);
        StartCoroutine(GetData()); // Start the coroutine to fetch data from the API
    }

    IEnumerator GetData()
    {
        // Create a GET request
        using (UnityWebRequest webRequest = UnityWebRequest.Get(getUrl))
        {
            yield return webRequest.SendWebRequest(); // Send the request and wait for response
            
            if (webRequest.result == UnityWebRequest.Result.Success)
            {
                Debug.Log("GET request successful. Response: " + webRequest.downloadHandler.text);
                // Display the response text in the UI
                responseText.text = "Response: " + webRequest.downloadHandler.text;
            }
            else
            {
                Debug.LogError("GET request failed. Error: " + webRequest.error);
                // Display error message if request fails
                responseText.text = "Error: " + webRequest.error;
            }
        }
    }
    
    // Method triggered when POST button is clicked
    public void PostRequest()
    {
        Debug.Log("Starting POST request to: " + postUrl + " with data: " + inputField.text);
        StartCoroutine(PostData(inputField.text)); // Start coroutine to send data to the API
    }

    IEnumerator PostData(string jsonData)
    {
        // Create a form and add input data
        WWWForm form = new WWWForm();
        form.AddField("data", jsonData);
        
        // Create a POST request with form data
        using (UnityWebRequest webRequest = UnityWebRequest.Post(postUrl, form))
        {
            yield return webRequest.SendWebRequest(); // Send request and wait for response
            
            if (webRequest.result == UnityWebRequest.Result.Success)
            {
                Debug.Log("POST request successful. Response: " + webRequest.downloadHandler.text);
                // Display the response received from the server
                responseText.text = "Response: " + webRequest.downloadHandler.text;
            }
            else
            {
                Debug.LogError("POST request failed. Error: " + webRequest.error);
                // Display error message if request fails
                responseText.text = "Error: " + webRequest.error;
            }
        }
    }
}

Ejercicio 3: Autenticación con Token y Manejo de Errores

Objetivo: Implementar un sistema de autenticación simple usando un token de acceso.

Instrucciones:

  1. Simula un inicio de sesión enviando un POST a localhost/api/login coN

    { "username": "Alumno123", "password": "clave123" }

  2. La API devolverá un token en la respuesta:

  3. Guarda el token y úsalo para futuras solicitudes GET añadiéndolo al header de la petición como Authorization: Bearer {token}.

Pistas:

Extras:

  • Implementar barra de carga mientras se hace la petición.

Glosario de Comandos y Métodos

1. Manejo de Corrutinas

  • StartCoroutine(GetData()); → Inicia una corrutina para ejecutar una tarea en paralelo (ideal para llamadas de red asíncronas).
  • IEnumerator GetData() → Define una corrutina que ejecuta código en múltiples frames sin bloquear el juego.
  • yield return webRequest.SendWebRequest(); → Espera la respuesta de la solicitud HTTP antes de continuar la ejecución.

2. Envío y Recepción de Peticiones HTTP

  • UnityWebRequest.Get(url); → Crea una solicitud GET para obtener datos de un servidor.
  • UnityWebRequest.Post(url, form); → Crea una solicitud POST para enviar datos al servidor.
  • webRequest.SendWebRequest(); → Envía la solicitud y espera una respuesta.
  • webRequest.result == UnityWebRequest.Result.Success → Comprueba si la solicitud fue exitosa.
  • webRequest.downloadHandler.text → Obtiene la respuesta del servidor en formato de texto.
  • webRequest.error → Devuelve un mensaje de error si la solicitud falla.

3. Manejo de Formularios

  • WWWForm form = new WWWForm(); → Crea un formulario para enviar datos mediante POST.
  • form.AddField("data", jsonData); → Agrega un campo con datos al formulario.

4. Depuración y Registros en la Consola

  • Debug.Log("Mensaje"); → Imprime un mensaje en la consola de Unity (para debug).
  • Debug.LogError("Mensaje de error"); → Imprime un mensaje de error en la consola de Unity.

5. UI en Unity

  • public InputField inputField; → Define un campo de entrada de texto en la interfaz.
  • public Text responseText; → Define un campo de texto donde se mostrará la respuesta.
  • responseText.text = "Response: " + webRequest.downloadHandler.text; → Actualiza el texto en la interfaz con la respuesta del servidor.

Archivo PHP para que maneje dos salidas en GET:

  • Texto plano si se pasa ?format=text en la URL.
  • JSON por defecto si no se especifica el formato.

Para POST:

  • Devolver JSON con los datos recibidos.
// Set default response header to JSON format
header("Content-Type: application/json");

// Define an associative array to store the response
$response = array();

// Determine the request method used by the client
$method = $_SERVER['REQUEST_METHOD'];

if ($method === 'GET') {
    // Check if a specific response format is requested in the URL parameters (e.g., ?format=json or ?format=text)
    $format = $_GET['format'] ?? 'json';
    
    if ($format === 'text') {
        // If 'text' format is requested, return a plain text response
        header("Content-Type: text/plain");
        echo "GET request received successfully!";
    } else {
        // Default response format is JSON
        $response["message"] = "GET request received successfully!";
        $response["status"] = 200;
        
        // Convert the response array into a JSON string and send it to the client
        echo json_encode($response);
    }
} 
elseif ($method === 'POST') {
    // Handling POST request
    // Retrieve data from the POST request, defaulting to "No data received" if not provided
    $postData = $_POST["data"] ?? "No data received";
    
    // Prepare response
    $response["message"] = "POST request received successfully!";
    $response["received_data"] = $postData; // Include the received data in the response
    $response["status"] = 200;
    
    // Convert the response array into a JSON string and send it to the client
    echo json_encode($response);
} 
else {
    // Handling unsupported request methods (e.g., PUT, DELETE, etc.)
    http_response_code(405); // Set HTTP status code to 405 (Method Not Allowed)
    $response["error"] = "Method not allowed";
    
    // Convert the response array into a JSON string and send it to the client
    echo json_encode($response);
}
