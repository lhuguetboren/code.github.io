    Introducción a React y TypeScript#

    ¿Qué es React?#

    React es una biblioteca de JavaScript utilizada para construir interfaces de usuario interactivas y eficientes. Fue creada por Facebook y es ampliamente usada en aplicaciones web modernas debido a su enfoque basado en componentes reutilizables.

    Características Clave de React#

    • Basado en Componentes: La UI se divide en pequeñas piezas reutilizables llamadas componentes.
    • Virtual DOM: React actualiza solo las partes necesarias del DOM, mejorando el rendimiento.
    • Unidireccionalidad de Datos: Control de flujo de datos predecible y estructurado.
    • Declarativo: Define la UI basada en el estado de la aplicación.
    • Gran Comunidad y Ecosistema: React tiene una gran cantidad de librerías y herramientas que facilitan el desarrollo.

    React con TypeScript#

    TypeScript es un superset de JavaScript que añade tipado estático, lo que ayuda a prevenir errores y mejorar la mantenibilidad del código. Integrarlo con React permite definir tipos en componentes, props y estados.

    Ventajas de usar TypeScript en React#

    • Mejor Autocompletado y Documentación en editores de código como VSCode.
    • Prevención de Errores al definir tipos estrictos para props y estados.
    • Código más Mantenible y fácil de refactorizar.
    • Mayor seguridad en la manipulación de datos en los componentes.

    Ejemplo: Componente con Props Tipadas#

    import React from "react";

interface SaludoProps {
  nombre: string;
}

const Saludo: React.FC<SaludoProps> = ({ nombre }) => {
  return <h1>Hola, {nombre}!</h1>;
};

export default Saludo;

    Explicación: - Se define una interfaz SaludoProps para tipar las props. - El componente Saludo recibe nombre y lo renderiza en pantalla.

    Manejo del Estado en React: useState#

    El hook useState permite manejar datos dinámicos dentro de un componente.

    Ejemplo de useState con TypeScript#

    import React, { useState } from "react";

const Contador: React.FC = () => {
  const [contador, setContador] = useState<number>(0);

  return (
    <div>
      <h2>Contador: {contador}</h2>
      <button onClick={() => setContador(contador + 1)}>Incrementar</button>
    </div>
  );
};

export default Contador;

    Explicación: - useState<number>(0): Se inicializa el estado con el tipo number. - setContador(contador + 1): Se actualiza el estado y React vuelve a renderizar el componente.

    Ciclo de Vida con useEffect#

    El hook useEffect permite ejecutar efectos secundarios en un componente.

    Ejemplo: useEffect para cargar datos desde una API#

    import React, { useState, useEffect } from "react";

const DatosAPI: React.FC = () => {
  const [datos, setDatos] = useState<string>("");

  useEffect(() => {
    fetch("https://jsonplaceholder.typicode.com/todos/1")
      .then((res) => res.json())
      .then((data) => setDatos(data.title));
  }, []);

  return <h3>Dato cargado: {datos}</h3>;
};

export default DatosAPI;

    Explicación: - useEffect(() => { ... }, []) ejecuta el código una sola vez al montar el componente. - Se obtiene un dato de una API y se almacena en el estado datos.

    Navegación con React Router#

    React Router es la librería más utilizada para manejar la navegación en aplicaciones React.

    Ejemplo: Configuración Básica de React Router#

    import React from "react";
import { BrowserRouter as Router, Routes, Route, Link } from "react-router-dom";

const Home: React.FC = () => <h1>Página de Inicio</h1>;
const About: React.FC = () => <h1>Acerca de Nosotros</h1>;

const App: React.FC = () => {
  return (
    <Router>
      <nav>
        <Link to="/">Inicio</Link> | <Link to="/about">Acerca</Link>
      </nav>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </Router>
  );
};

export default App;

    Explicación: - Se usa BrowserRouter para habilitar la navegación en la aplicación. - Routes y Route definen qué componente se renderiza en cada URL. - Link reemplaza las etiquetas <a> para evitar recargar la página.

    Herramientas de Depuración en React#

    Para mejorar la productividad, React ofrece varias herramientas de depuración: - React Developer Tools: Inspecciona componentes, props y estado en tiempo real. - Console.log y Debugger: Para analizar valores y ejecución del código. - ESLint y Prettier: Mejoran la calidad y consistencia del código.

    Ejemplo: Uso de console.log y debugger#

    const handleClick = () => {
  console.log("Botón presionado");
  debugger; // Detiene la ejecución en DevTools
};

    ESLint en React#

    ESLint es una herramienta que ayuda a mantener un código limpio y sin errores en proyectos JavaScript y TypeScript.

    graph TD;
    A[ESLint] -->|Detecta errores| B[Errores de Sintaxis]
    A -->|Mejora Calidad| C[Buenas Prácticas]
    A -->|Formateo Automático| D[Prettier]
    • Errores de Sintaxis: Detecta problemas en el código antes de ejecutarlo.
    • Buenas Prácticas: Evita malas prácticas de programación y código obsoleto.
    • Formateo Automático: Se puede integrar con Prettier para un formato de código consistente.

    Configuración de ESLint#

    Después de la inicialización, puedes modificar .eslintrc.js para adaptarlo a tu proyecto:

    module.exports = {
    env: {
        browser: true,
        es2021: true
    },
    extends: [
        "eslint:recommended",
        "plugin:react/recommended",
        "plugin:@typescript-eslint/recommended"
    ],
    parser: "@typescript-eslint/parser",
    parserOptions: {
        ecmaFeatures: {
            jsx: true
        },
        ecmaVersion: "latest",
        sourceType: "module"
    },
    plugins: ["react", "@typescript-eslint"],
    rules: {
        "react/prop-types": "off",
        "indent": ["error", 4],
        "quotes": ["error", "double"],
        "semi": ["error", "always"]
    }
};

    Para ejecutar ESLint en tu código:

    npx eslint src --fix

    Esto revisará el código y corregirá errores automáticamente.
