    E useeeffect solucion

    ✅ Ejercicio 1: Diagnóstico de Comportamiento#

    const ContadorEstatico: React.FC = () => {
  const [contador, setContador] = useState(0);

  useEffect(() => {
    setTimeout(() => {
      setContador(contador + 1);
    }, 1000);
  }, []);

  return <h1>Contador: {contador}</h1>;
};

    🔍 Explicación paso a paso:#

    const [contador, setContador] = useState(0); // Declaramos un estado inicial con valor 0.

    useEffect(() => {
  setTimeout(() => {
    setContador(contador + 1); // Intentamos incrementar el contador después de 1 segundo.
  }, 1000);
}, []);
    • El useEffect se ejecuta una sola vez al montar el componente, por el array vacío [].
    • El setTimeout se programa para dentro de 1 segundo.
    • Dentro del setTimeout, usamos contador + 1, pero React cierra sobre el valor inicial, que es 0 → por tanto siempre suma 0 + 1.

    🔴 Resultado: Se mostrará "Contador: 1" después de 1 segundo, pero nunca aumentará más.

    📈 Mejora posible: Usar función actualizadora para evitar problemas de closure:

    setContador(prev => prev + 1);

    🛠️ Ejercicio 2: Construcción guiada con condición oculta#

    const Monitor: React.FC = () => {
  const [valor, setValor] = useState(0);

  useEffect(() => {
    const intervalo = setInterval(() => {
      setValor((prev) => (prev % 4 === 0 ? prev + 2 : prev));
    }, 2000);

    return () => clearInterval(intervalo); // Limpieza del intervalo al desmontar o actualizar.
  }, []); // Dependencias vacías = solo se monta una vez.

  return (
    <div>
      <h1>Valor: {valor}</h1>
      <button onClick={() => setValor(0)}>Reiniciar</button>
    </div>
  );
};

    🧠 Explicación por pasos:#

    • Estado inicial: valor = 0.
    • Cada 2 segundos: prev % 4 === 0 → si es divisible entre 4, se incrementa en 2.
    • Si el valor no es múltiplo de 4, se mantiene igual.
    • Botón permite reiniciar manualmente.

    📌 Clave didáctica: Usar el valor anterior dentro del setValor con función callback es más robusto que depender de valor directamente.

    🧪 Reflexión sobre dependencias:

    • []: el useEffect no se vuelve a ejecutar si valor cambia. Es correcto porque usamos función en setValor.
    • [valor]: generaría un nuevo intervalo cada vez que cambia el valor, acumulando intervalos → error común.

    🌐 Ejercicio 3: Simulación asincrónica con dependencias dinámicas#

    const UsuarioAleatorio: React.FC = () => {
  const [categoria, setCategoria] = useState("admin");
  const [usuario, setUsuario] = useState<string | null>(null);

  useEffect(() => {
    let cancelado = false;     // Bandera para cancelar si hay cambio de categoría
    setUsuario(null);          // Indicamos "Cargando..."

    const timeout = setTimeout(() => {
      if (!cancelado) {
        setUsuario("Usuario " + Math.floor(Math.random() * 1000));
      }
    }, 1000);

    return () => {
      cancelado = true;        // Si la categoría cambia, cancelamos este efecto anterior
      clearTimeout(timeout);   // Eliminamos el timeout anterior para evitar conflictos
    };
  }, [categoria]); // Se vuelve a ejecutar cada vez que cambia la categoría

  return (
    <div>
      <select value={categoria} onChange={(e) => setCategoria(e.target.value)}>
        <option value="admin">Admin</option>
        <option value="editor">Editor</option>
        <option value="viewer">Viewer</option>
      </select>

      <p>{usuario ? `Usuario: ${usuario}` : "Cargando..."}</p>
    </div>
  );
};

    🧠 Explicación:#

    • Cada vez que cambia categoria:
    • Se borra el usuario actual (setUsuario(null)).
    • Se programa un setTimeout para simular una carga de red.
    • Si el efecto anterior aún no ha terminado, lo cancelamos manualmente.

    🛡️ Técnica clave: cancelado = true junto con clearTimeout → evita condiciones de carrera y actualizaciones de estado incorrectas en efectos asincrónicos.
