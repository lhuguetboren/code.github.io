    Aplicació REACT#

    1. ✅ WebSockets per comunicació en temps real
    2. 🔄 Comunicació síncrona (HTTP) per peticions clàssiques amb fetch o axios
    3. 📁 Pujada de fitxers mitjançant formulari amb multipart/form-data

    📦 Esquema del projecte React#

    react-app/
│
├── public/
│   └── index.html
│
├── src/
│   ├── components/
│   │   ├── WebSocketChat.jsx        # Component de xat en temps real amb WebSocket
│   │   ├── SyncRequest.jsx          # Component per fer peticions síncrones amb fetch
│   │   ├── FileUpload.jsx           # Formulari per pujar fitxers
│   │
│   ├── services/
│   │   ├── websocketClient.js       # Gestor de connexió WebSocket
│   │   ├── api.js                   # Funcions de peticions HTTP (fetch o axios)
│   │
│   ├── App.jsx                      # Arrel de la UI
│   └── main.jsx                     # Entrada del projecte (ReactDOM)
│
├── server/
│   ├── server.js                    # Backend amb Node.js (WebSocket + API + Upload)
│   └── uploads/                     # Carpeta per guardar els fitxers pujats
│
├── .gitignore
├── package.json
└── README.md

    🔌 Tecnologies previstes#

    • Frontend (React):
    • React (amb Vite o Create React App)
    • WebSocket natiu per a temps real
    • fetch o axios per comunicació síncrona

    • FormData per pujada de fitxers

    • Backend (Node.js):

    • Express per peticions síncrones i pujada de fitxers
    • websocket o ws per connexions WebSocket
    • multer per gestionar multipart/form-data

    🧠 Flux de treball per cada funcionalitat:#

    1. WebSocket#

    • El client connecta a ws://localhost:8089
    • S’envien i reben missatges entre usuaris en temps real

    2. Comunicació síncrona (HTTP)#

    • GET i POST a /api/*
    • Es fan peticions amb fetch i es mostra el resultat

    3. Pujada de fitxers#

    • Formulari que permet seleccionar i pujar un fitxer
    • El servidor el desa a /uploads i respon amb info
